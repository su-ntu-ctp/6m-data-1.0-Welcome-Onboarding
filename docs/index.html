🟢 Phase 1: The Foundation

Lessons 1.0 – 1.2

Build the mental model for what data science is, why databases exist, and how data is structured — before writing a single line of SQL or Python.

Lesson 1.0

Welcome & Onboarding

Set up your tools, join Discord, meet the cohort.

🏆 Hackathon: Understand the module structure and what the hackathon will ask of you.

Lesson 1.1

Intro to Data Science

Analytics → Data Science → AI hierarchy; data pipelines (Collection → Cleaning → Analysis → Viz); data ethics & bias.

🏆 Hackathon: Frame the kinds of questions your dataset can answer and identify the right analytical approach.

Lesson 1.2

Data Modeling & Schema Design

Relational vs. NoSQL vs. Vector databases; ERDs; Primary & Foreign Keys; Normalization (1NF–3NF).

🏆 Hackathon: Read and interpret the dataset's schema — understand how tables connect before writing a single query.

Coaching Session 1

Tools & AI Companion

🗂️ Git & GitHub

Create a repo, commit your work, push to GitHub. Every hackathon deliverable will be version-controlled from this point on.

📚 Google NotebookLM

Upload course materials and use NotebookLM to summarise content, quiz yourself, and generate study notes.

💬 Effective Prompting

Write clear, specific prompts to debug code, explain concepts, and brainstorm analysis angles with AI tools.

No hackathon deliverable this session — focus entirely on getting comfortable with the tools.

🔵 Phase 2: SQL — Querying the Warehouse

Lessons 1.3 – 1.5

You cannot analyse a large dataset in Excel. SQL lets you ask precise questions of millions of rows in seconds — and connects directly to the hackathon dataset your group will investigate.

Lesson 1.3

SQL Basic — DDL

CREATE TABLE, data types, constraints (PK/FK/NOT NULL/CHECK), COPY for bulk import, indexes, views.

🏆 Hackathon: Create the database tables and load the hackathon dataset.

Lesson 1.4

SQL Basic — DML

SELECT, WHERE, ORDER BY, aggregates (COUNT/SUM/AVG/MIN/MAX), GROUP BY, HAVING, CASE, CAST.

🏆 Hackathon: Write the first exploratory queries — "Which category has the highest average value? What is the distribution by region?"

Lesson 1.5

SQL Advanced

JOINs (Inner/Left/Right/Full), UNION, window functions (running totals, RANK), Common Table Expressions (CTEs).

🏆 Hackathon: Combine multiple tables, rank records, and build the complex queries your analysis needs.

Coaching Session 2

Hackathon Kickoff & SQL Analysis

👥 Form Groups

Teams of 3–4 learners. Choose a dataset domain together (e.g. retail, healthcare, transport, finance).

🚀 Hackathon Kickoff

Each group defines 3–5 business questions to answer with the dataset. Questions are reviewed with the coach before analysis begins.

🔍 SQL Analysis Sprint

Use SELECT, GROUP BY, JOINs, and at least one window function to start answering your questions.

📤 Deliverable: A .sql file with your group's exploratory queries and a brief written summary of what you found. Share in #peer-reviews on Discord.

🟠 Phase 3: Python — The Analytical Engine

Lessons 1.6 – 1.7

SQL retrieves the data. Python transforms, calculates, and models it at scale. This phase introduces the two core Python libraries that underpin every data science workflow.

Lesson 1.6

Intro to NumPy

ndarray creation, broadcasting, indexing & slicing, vectorised operations, basic linear algebra. Why NumPy is 100x faster than Python loops on large datasets.

🏆 Hackathon: Perform fast numerical calculations on dataset columns — percentage changes, normalisation, statistical aggregations.

Lesson 1.7

Intro to Pandas

DataFrames & Series, .loc / .iloc, Boolean filtering, .apply(), sort & rank.

🏆 Hackathon: Load the dataset into Python, filter it, and begin exploring it programmatically.

Coaching Session 3

Python Analysis

📂 Load & Inspect

Load your dataset into a Pandas DataFrame. Run .info(), .describe(), .value_counts() on key columns.

🔎 Filter & Slice

Reproduce your SQL findings using Pandas Boolean filters and .groupby().

🔢 NumPy Calculations

Use NumPy for statistical or numerical calculations — normalising values, calculating percentage changes.

📤 Deliverable: A Jupyter notebook with structured sections, readable code, and brief written interpretations of each output. Share your notebook link in #peer-reviews on Discord.

🟣 Phase 4: EDA — Finding the Insights

Lessons 1.8 – 1.9

Real data is messy. This phase turns raw data into trustworthy, analysis-ready insights — cleaning problems before they poison your conclusions, then aggregating to surface the patterns that matter.

Lesson 1.8

EDA Basic

The data health check: .info(), .describe(), .value_counts().

Handling missing values (fillna / dropna), duplicates, outlier detection (IQR, z-score).

Type conversion, string cleaning, categorical encoding, file I/O (CSV, JSON, Excel).

🏆 Hackathon: Clean your dataset — fix types, remove nulls and duplicates — so every downstream result can be trusted.

Lesson 1.9

EDA Advanced

Datetime parsing, DatetimeIndex, resampling (.resample()), rolling windows.

SQL-style merges with pd.merge(); wide ↔ long reshaping (melt / pivot).

GroupBy (Split-Apply-Combine), pivot_table(), crosstab().

🏆 Hackathon: Aggregate by time period, merge supporting tables, and generate the summary statistics your presentation will be built on.

🔴 Phase 5: Visualisation & Storytelling

Lesson 1.10

Analysis is only as valuable as its communication. This phase teaches you to turn numbers into decisions — by choosing the right chart, applying design principles, and framing your findings as a three-act story.

Lesson 1.10

Data Visualisation & Storytelling

👁️ Perception

Pre-attentive processing; cognitive load; how to make insights jump out within 500ms.

🎨 Design

Chart selection rules; honest visualisation (bar charts start at 0); Matplotlib Figure/Axes/Plot; Seaborn statistical graphics.

📖 Storytelling

Three-act structure (Context → Insight → Action); audience-first communication; executive vs. technical framing.

🏆 Hackathon: Build the charts that support your group's narrative and frame them in a clear three-act story for the final presentation.

Coaching Session 4 — Final

EDA Wrap-Up & Hackathon Presentations

🔬 EDA Wrap-Up (first half)

Finalise your cleaning, pivot tables, and visualisations. Coaches circulate to help groups that are stuck.

🎤 Group Presentations (second half)

Each group presents for 10 minutes including Q&A — roughly 7 minutes to present, 3 minutes for questions from the cohort and coaches.

📋 Presentation must include:

  • ✓ The business questions your group chose to investigate
  • ✓ A summary of the dataset and any data quality issues found
  • ✓ At least 3 visualisations with clear titles and axis labels
  • ✓ 3 or more data-backed insights stated in plain language
  • ✓ One recommendation a non-technical decision-maker could act on

📤 Upload your final notebook and any slides to #peer-reviews on Discord before the session starts.

🏆 Mini-Hackathon

A progressive group project woven through the entire module

The Mini-Hackathon is not a single event. It builds across all 4 coaching sessions — so by the time you present, your notebook is already 80% done and there is no last-minute scramble.

📊 Presentations are evaluated on four dimensions

🎯

Relevance

Do your business questions matter? Would a real decision-maker care about the answer?

🔬

Rigour

Is the data clean? Can the numbers be trusted? Were data quality issues handled correctly?

💡

Clarity

Would a non-technical audience understand your insights without reading the code?

📖

Storytelling

Do the visualisations support a clear narrative — context, insight, and recommendation?
